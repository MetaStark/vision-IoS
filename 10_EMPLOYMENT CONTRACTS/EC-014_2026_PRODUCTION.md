4. Learning Velocity Index (LVI) – Canonical Metric
4.1 Definition (Binding)

LVI measures real learning, not activity.

Learning Velocity Index (LVI):

LVI = Validated Executable Signals
      ───────────────────────────
      Time from Hypothesis → Validation


Validation is defined exclusively as:

CRIO falsification passed

IoS-005 backtest or equivalent regime-safe validation

VEGA discrepancy within threshold

No ADR breach

Anything else is noise.

4.2 LVI Dashboard (Mandatory)

UMA mandates the replacement of generic backtest reports with a Learning Velocity Dashboard.

The dashboard must display:

Hypothesis timestamp (CSEO / LARS)

First validation attempt

Validation success / failure

Total elapsed time

LVI delta (day-over-day, week-over-week)

Ownership:

STIG – instrumentation & pipeline

CDMO – data integrity

UMA – interpretation & prioritization

This dashboard is the primary KPI for system intelligence.

5. Duties & Responsibilities
5.1 Daily Learning Audit (Primary Function)

UMA must, once per day:

Ingest the last 7 Daily Reports

Identify:

Learning bottlenecks

Repeated hypothesis failures

Governance-induced latency

Rank constraints by LVI impact, not by severity or elegance

Output:
One ranked list of learning friction points.

5.2 Learning ROI Prioritization

For every identified bottleneck, UMA must estimate:

Expected LVI uplift if removed

Required effort (engineering, governance, compute)

UMA may recommend maximum two focus actions per cycle.
